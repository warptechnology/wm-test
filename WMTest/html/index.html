Задачи для C# + СУБД SQL Server или ORM

Решить задания 2-мя способами:
  • C# + ORM (EF, NHibernate, и т.п.);
  • C# + хранимые процедуры.

Демонстрацию решений представить в виде:

веб сервисна выбор
  • WebServices
  • WCF
  • WebApi

клиент к созданному сервису (в любом виде), предусмотреть создание условий, гонок при обращении к сервису; скрипт создания и инициализации базы данных. Будет плюсом, если при создании демонстрационного приложения будут продемонстрированы знания ООП, SOLID, шаблонов проектирования.

Задача 1

Есть таблица (id int not null identity primary key, name varchar (50) unique). Нужно реализовать алгоритм GetOrAdd - получение существующей или добавление новой записи по имени, с возвратом идентификатора. Предусмотреть одновременное (возможно с разных компьютеров) обращение к GetOrAdd, т.е. не должно быть Unique Key Violation.

Задача 2

Есть таблица (id int not null primary key, value int not null). Нужно реализовать алгоритм AddOrUpdate - обновить существующее значение или вставить новое. Предусмотреть одновременное (возможно с разных компьютеров) обращение к AddOrUpdate. Исключить deadlock, unique key violation при параллельном вызове. Использование TABLOCK нежелательно.

Задача 3

Есть таблица (id int primary key, balance money). Нужно реализовать алгоритм перевода Transfer - первод средств с id1 на id2. Предусмотреть одновременное (возможно с разных компьютеров) обращение к Transfer. Исключить deadlock, unique key violation при параллельном вызове. Использование TABLOCK нежелательно.
